Core Problem

Mahasiswa STT Nurul Fikri kesulitan mengakses informasi beasiswa karena data tersebar di berbagai platform (IG, WA, Web Kampus). Hal ini menyebabkan inefisiensi pencarian dan tingginya angka "Missed Deadline".

Proposed Solution

Membangun **Portal Informasi Beasiswa Terpusat** (MVP) yang memfasilitasi agregasi link, filter kriteria akademik (IPK/Semester), notifikasi pengingat, dan dashboard tracking manual.

Measurable Organizational Value (MOV)

01

Better

  • Akurasi data terverifikasi admin.
  • Skor Kepuasan UI/UX ≥ 4.0/5.0.
02

Faster

  • Reduksi waktu cari beasiswa 50%.
  • Loading Page < 3 detik.
03

Do More

  • Partisipasi beasiswa naik 30%.
  • Tracking progres secara mandiri.

3-Tier Layered Design

Implementasi SAD menggunakan arsitektur 3-Tier berbasis Laravel untuk pemisahan tanggung jawab yang jelas:

P

Presentation Layer

Blade Templates, Tailwind CSS, Vite.

L

Logic Layer (Application)

Laravel Controllers, **Service & Repository Pattern**.

D

Data Layer

MySQL 8.0, Eloquent ORM, Migrations.

Database Business Rules (ERD Specs)

Data Integrity

**Soft Deletes**: Tabel `scholarships` menggunakan histori agar data tidak hilang permanen.

**Unique Slugs**: SEO friendly URL dan pencegahan data ganda pada tabel beasiswa.

Performance

**Indexing**: Kolom `slug`, `email`, dan `deadline` wajib di-index untuk optimasi query filter.

**On Delete Restrict**: Kategori yang memiliki data beasiswa tidak boleh dihapus.

Git Workflow (Gitflow)

M

Main Branch

Hanya untuk kode produksi stabil. Penggabungan wajib melalui PM/SM review.

D

Develop Branch

Branch utama harian. Semua fitur baru di-merge ke sini sebelum rilis.

F

Feature Branches

Format: `feature/[nama-fitur]`. Gunakan Pull Request (PR) untuk merge.

Naming Convention

Controller ScholarshipController
Model Scholarship
Table scholarships (plural)
Service ScholarshipService

Service Object Pattern

"Jangan menaruh logika bisnis yang kompleks di Controller."

  • Pindahkan logika filter & search ke `app/Services`.
  • Gunakan `FormRequest` untuk validasi input.
  • Maintainable, Testable, & SOLID compliant.
